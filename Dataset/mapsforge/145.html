From thilo.mu...@gmail.com on July 15, 2011 10:39:56

Comparing .map files with .pbf files does not make that much sense, as our binary map files are optimized for fast (tile based) rendering. This requires some level of redundancy and index structures which obviously increases the file size. PBF is neither optimized for rendering nor for anything else (searching, routing, ...), it just serves as a compact container format for lossless data exchange.

Nevertheless, i agree that your examples show cases in which the file size is larger than necessary. This might also be caused by bugs in the mapfile writer. We will work on that, but i cannot make any promise what we will achieve. There are also efforts to replace the binary map file format with an embedded database, maybe one day this will provide an even better solution for the problem.

From thilo.mu...@gmail.com on December 23, 2011 13:47:06

With the latest version of the mapfile-writer, we are now clipping all ways and filtering out even more way points. In combination with other improvements in the new version of the binary file format (like the optional double-delta encoding of way points), this will make our map files smaller.

When the 0.3.0 release has been published, we should check again if the mentioned figures for Bolivia and Libya are still valid, otherwise we could then close this issue.
