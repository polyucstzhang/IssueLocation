From thilo.mu...@gmail.com on December 18, 2011 14:40:58

The SVN version is under heavy development and neither fully tested nor documented. It is recommended to use the last official release until we publish version 0.3.0 - which will hopefully take place in only a few weeks. We have just switched our build process from Ant to maven, therefore the information in our wiki is outdated.

If you really want to try out the latest SVN version, here is a quick how-to (tested only on Linux, not Windows):

1) Check out the HEAD version of the trunk/mapsforge-map directory via SVN
2) Open a command line and go to the mapsforge-map directory
3) Run "mvn clean install" and wait until everything is finished

This should generate a new file called "mapsforge-map-0.3.0-SNAPSHOT.jar" in the "target" subdirectory, which can be used in Android applications like the AdvancedMapViewer.

As the file format of our map files has changed a lot since the last 0.2.4 release, you will have to create your own map file using the latest SVN version of the mapfile-writer. There are a few test files on our download server ( http://download.mapsforge.org/maps/v3-test-maps/ ) which might also work. Note that the file format is not yet finalized and might undergo further changes.

We know that the current situation is unpleasant for users of our software. We are working hard towards our next release, which will be accompanied by an update of our documentation. Until that is done, thank you for your patience.
