From trohwe...@gmail.com on June 04, 2014 13:39:04

The line

return Thread.currentThread().getClass().getResourceAsStream(this.absolutePath + this.file);

in org/mapsforge/map/rendertheme/InternalRenderTheme

does not make much sense and does not work as expected depending on
the classloader setup. Thread.currentThread().getClass() gives you
most likely just Thread.class! But you want your resources resolved with respect to
the classloader of some application class, not the classloader of some arbitrary system class. I suggest replacing this by

getClass().getResourceAsStream(this.absolutePath + this.file);

i.e. resolving the resources with respect to InternalRenderTheme.class
This works for example when putting the resources together with the rendertheme classes as library file in android. The original code does not work.

Original issue: http://code.google.com/p/mapsforge/issues/detail?id=503
