FileOpenResult "invalid file size" error #547

Closed
jordantrain opened this Issue Nov 21, 2014 · 5 comments

None yet

Milestone

No milestone

Assignee

No one assigned

3 participants

jordantrain commented Nov 21, 2014

I recently added the Mapsforge API v. 0.4 to my android app and am receiving crash reports with the following error:

... Caused by: java.lang.IllegalArgumentException: invalid file size: 59214803
at org.mapsforge.map.layer.renderer.TileRendererLayer.setMapFile(TileRendererLayer.java:90)
at ...

The error is thrown inside my onStart() method where I attempt to set the map file.

My onStart method has:

           this.mapView.getModel().mapViewPosition.setZoomLevel((byte) zoomLevel);

            this.tileRendererLayer = new TileRendererLayer(tileCache,
                            this.mapView.getModel().mapViewPosition, false, AndroidGraphicFactory.INSTANCE);
            tileRendererLayer.setMapFile(getMapFile());

This appears to be a very infrequent event and I am unable to replicate the error on any of my test devices.

My onStop and onDestroy methods do some cleanup: 

    @Override
    protected void onStop() {
            super.onStop();
            this.mapView.getLayerManager().getLayers().remove(this.tileRendererLayer);
            this.tileRendererLayer.onDestroy();

    }

    @Override
    protected void onDestroy() {
            super.onDestroy();
            this.tileCache.destroy();
            this.mapView.getModel().mapViewPosition.destroy();
            this.mapView.destroy();
            AndroidResourceBitmap.clearResourceBitmaps();
    }

Any incites or solutions would be greatly appreciated!

ludwigb commented Nov 21, 2014
You do not say anything about the wider context, but if your app uses a selectable file (such as a map downloaded from the internet) then it is possible that a user tries to open an incomplete map file (the routine compares the file size on disk with the map size encoded in the header). Ludwig
