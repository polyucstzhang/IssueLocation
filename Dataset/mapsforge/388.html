From jjbross@gmail.com on December 12, 2012 04:35:32

Thanks for reporting this issue.

When writing the binary map file format, we internally use some buffers to cache individual writes and then batch write a complete buffer to the disk (increases
disk write performance). The problem indicates that one of the internal buffers exhibits an overflow. Theoretically, you could increase the size of the buffer (currently 10MB, defined by a constant in source code) to get rid of the exception and write a proper map file. However, the result would be very large data tiles (> 10MB) in the map file which cannot be handled adequately by the renderer. Most probably, you would receive an OutOfMemoryException when loading the map.

To solve the problem, you must either simplify the data, for example less contour lines, or you can tweak the size of the data tiles (see option "zoom-interval-conf"). I suggest to do both.

Again yes. If zoom-apper=12 for contourlines, then they only affect the data size of tiles within your third interval. Your conclusion is correct, they cannot be related to the mentioned issue.
