mapsforge commented Nov 6, 2014

From thilo.mu...@gmail.com on February 20, 2012 11:07:59

The answer is simple: you are using a wrong map file. With the upcoming 0.3.0 release, we have changed the binary file format, see the wiki article SpecificationBinaryMapFile for details. Version 2 was used in the previous 0.2.4 version, but the latest code in our SVN supports only version 3.

You can check out the latest version of the map-writer and create a file with the new format yourself. Some ready-to-download files can also be found on our download server: http://download.mapsforge.org/maps/snapshot/

From jjbross@gmail.com on February 21, 2012 04:53:26

The exception says that osmosis cannot find the plugin. Please make sure that the correct jar file is in one of the recognized plugin directories. See the osmosis documentation of what is a valid plugin directory: http://wiki.openstreetmap.org/wiki/Osmosis/Detailed_Usage#Plugin_Tasks .openstreetmap/default/plugin seems to be an invalid directory!

How did you build the map-file-writer*.jar? Normally the build process should copy the correct jar to the ~/.openstreetmap/osmosis/plugins directory. The maven build also produces a jar file that includes all dependencies, so no need to install trove.
