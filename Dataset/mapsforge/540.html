I have pushed a Maven change on dev.

DocLint is a new feature in Java 8 which provides a means to detect errors in Javadoc comments early in the development cycle and in a way that is easily linked back to the source code.

It appears we have 3 solutions:

  • Fix the comments (I prefer this).
  • Disable DocLint, with -Xdoclint:none but javadoc <8 doesn't know this option. The formal way would be to use profiles in the build.
  • Ignore DocLint exit codes.

For now I followed the 3rd solution, please report any issues.

I have seen reports that the Gradle also may needs adjustments?
