From LudwigBr...@gmail.com on April 11, 2011 06:48:09

I know there is work going on on this issue, but I think the issue calls for a simple solution in itself and perhaps in addition to the XML themes. I see this as complementary to the XML scheme.

My solution here simply adds a variable for the font-scale into the DatabaseMapGenerator which can be changed at runtime, resulting in larger fonts and wider roads to accommodate the larger font. It provides a method on the MapView that stops the tile generation machinery, changes the font size, and clears the cache and restarts the tile generation. The change in setting can be triggered from within the preferences, so when a user returns to the map screen the new font setting will take immediate effect with currently visible tiles regenerated. The code to trigger this from the preferences is not included here (it could be simply a check in the onResume in the MapViewer that will effect the change, I will supply the code on request).

I have attached two patches: one implements the cache clearance, which might be useful on its own, the other patch affects the MapView and the DatabaseMapGenerator. While this is not extensively tested it seems to work.

I am aware that simply enlarging the fonts and making the roads wider does somehow upset the map layout, however, with factors chosen between 1.0 (the current size) and 2.0 the effect is not too disturbing and probably preferential for someone who otherwise cannot read street names.

I have attached a few screenshots that show the effect for font scale factors of 1.0, 1.4 and 2.0. The changes in the patches are backward compatible and without a change in the scale factor the map will just look like before.

Attachment: cache.patch font.patch detail-10.png detail-14.png detail-20.png font1.0.png font2.0.png

From thilo.mu...@gmail.com on April 16, 2011 09:36:00

Finally i had some time to look at your patch. I like your solution for its simplicity and high benifit, however i implemented it a little bit different. First i disapprove the idea of clearing the cache, as this kills a lot of work and spent CPU time. Second i don't like changing the stroke widths together with the font size, as the default values have been choosen carefully. Modifying them will in most cases decrease the visual quality of the map (for example overlapping streets).

Based on your patch i have made some changes (see revision r1005 ) to add a setTextScale(float) method to the MapView. I also added an entry to the preferences menu of the AdvancedMapViewer. Please check out the new version and let me know if you are satisfied with the result. If you agree we can close this issue. :-)
