ZXing landing page fails to handle manual entry #186

Closed
l-dobrev opened this Issue Jul 15, 2014 · 6 comments

Labels

None yet

Milestone

  3.2.0

Assignee

srowen

2 participants

l-dobrev commented Jul 15, 2014

Open
http://zxing.appspot.com/scan?SCAN_FORMATS=QR_CODE&ret=http%3A%2F%2Fgoogle.com%3Fq%3D{CODE}

Attempt to manually specify the code
123

Result:
http://zxing.appspot.com/http%3A%2F%2Fgoogle.com%3Fq%3D%7BCODE%7D?&{CODE}=123

Expected:
http://google.com/?q=123

l-dobrev commented Jul 15, 2014

I failed to prepare a Pull Request,
The solution should be relatively simple:
In
zxing.appspot.com/src/main/webapp/scan.html:52
replace
GET[prm[0]] = prm[1];
with
GET[decodeURIComponent(prm[0])] = decodeURIComponent(prm[1]);
