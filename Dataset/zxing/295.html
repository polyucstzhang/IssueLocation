smilog commented Jan 22, 2015

We use this in ZXING in with our data collection applications for warehouse logistics.
Often we scan barcodes that conform to the GS1 standard, that contain multiple fields.
For variable length fields, a FNC1 character (ascii 29) is used as a terminator.
For 1D Code128 formated barcodes, the FNC1 character is suppressed.

wfred101 commented Mar 7, 2015

ISO standards demand the following behavior:
FNC1 at the first position - This is tranlated to the symbology identifier according ISO/IEC 15424. A Code 128 for example get ]C0 as identifier if there was no FNC1 at the first position. If there was one then the symbology identifier gets ]C1. There is no other translation of FNC1 if it is at the first position. In case the FNC1 is beeing use at another position (except second) as field separator then the ISO standards demand that the FNC1 character shall be translated to the GS character ASCII 29.

The Code 128 (ISO/IEC 15417) modifiers
0 a normal Code 128 - no FNC characters at the first or second position
1 is a FNC1 at the first position. GS1 call then this Code 128 "GS1 128"
2 is a FNC1 charater at the secon position
4 is a concantenated symbol according to ISBT demands
