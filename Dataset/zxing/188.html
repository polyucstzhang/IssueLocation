srowen commented Jul 15, 2014

That's weird. I can think of two possible reasons off the top of my head. One is pretty unlikely: that the images get read slightly differently because underneath Java is delegating to different native libraries for image parsing. This happened a long time ago with JPEGs on Java 5 vs 6, but thought we fixed that by switching to PNGs. I think it might explain a few failures but not that many.

I also wonder if your checkout of the files containing the correct barcode contents changed things like line separators. That might cause it to mismatch the expected contents Again still seems a bit unlikely.

It all passes on Java 6, 7, and 9 on Travis CI (Linux of course).

I'm not really sure but whatever it is is likely Windows specific. If you have any ideas that would make it work everywhere let me know. Otherwise not sure what to do.

mmhelm commented Jul 16, 2014

I've just ignored the QRCodeBlackBox5TestCase with @Ignore and the build passed. So it looks like the problem is isolated to the QRCodeBlackBox5TestCase.
